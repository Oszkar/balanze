## 1. Goal & scope

Ship the **glanceable tray popover** that renders the (already solid, dashboard-accurate) data layer, plus the **color-shifting gauge tray icon** — and wire the live `watcher → state_coordinator → TauriSink` spine into the Tauri host for the first time (the v0.2 Sink-seam, exercised live; the #1 remaining roadmap risk).

### In scope (v0.3.0)

- Tauri 2 popover UI (Svelte 5) rendering the `Snapshot`: **Grid + Cards** views with a density toggle, light/dark theme following the OS, hover provenance, degraded-visible markers, the subscription-leverage box.
- Live backend boot inside the Tauri host: `StateCoordinator` + supervised `Watcher` tasks + `TauriSink`.
- IPC: commands `get_snapshot`, `refresh_now`; events `usage_updated`, `degraded_state`.
- **Gauge-ring** tray icon (worst-window color-shift), repaint deduped per §3.1.
- Adaptive layout for 0 / 1 / 2 connected providers.
- `tauri-plugin-single-instance` (already wired in the scaffold).

### Out of scope (deferred)

| Deferred item | Lands in |
|---|---|
| 3-style tray picker (Text glance / Minimal mark) + the style setting | v0.3.1 |
| Settings UI, `keyring`→`keyring-core` migration, in-app "Connect →" | v0.3.1 |
| `set_api_key` / `get_settings` / `set_settings` commands | v0.3.1 |
| `get_history` command, **burn sparkline**, SQLite history | v0.3.3 |
| Alerts / OS notifications | v0.3.2 |
| Dashboard window | v0.3.3 |

Rationale for the `get_history`/sparkline deferral: the only v0.3.0 consumer of `get_history` is the sparkline, and the sparkline needs a burn *series* the `Snapshot` does not keep (only the scalar `recent_burn_tokens_per_min`). Building an IPC command with no reader is YAGNI; the sparkline + durable history + `get_history` ship together in v0.3.3. v0.3.0 shows the burn **number** where the sparkline will later sit.

## 2. The locked visual model

Decided via visual brainstorming on top of `Balanze Wireframes.html`.

- **Skin:** one theme that **follows the OS (light + dark)** via `prefers-color-scheme`, in a *refined* (not hand-drawn) character. Color is used **strictly for status** (`ok` green / `warn` amber / `bad` red); the wireframe's earthy palette is the reference (`--ok #3f8f5f` / `--warn #cf8a2a` / `--bad #c0493a`; provenance accent `--real #3667a6`, tuned brighter in dark).
- **One surface, density toggle:** `▦ Grid` ⇄ `▤ Cards` (segmented control in the header). Default `Grid`.
- **Grid** (the compact view): **transposed matrix** — providers as **columns** (Anthropic, OpenAI — Claude Code & Codex weighted equally), metrics as **rows** (quota, billed). **No row-label gutter and no per-cell badges** — the value format (`62%` vs `$12.74`) self-identifies the row, and the column headers + leverage box carry the structure. Each quota cell: big `%` + a `UsageBar` where **fill = used %, tick = window-elapsed %** (pace inline, no separate bar) + reset clock + secondary window. Billed cell: `$` amount or `unavailable` (hatched). **Hover any cell → source + confidence** tooltip.
- **Cards** (the roomier view): one card per provider, **one `UsageBar` per cadence window** (incl. per-model variants like `Sonnet · 7-day`), and **keeps inline `REAL $` / `N/A` badges** (it has the room — density-differentiated provenance: Grid = minimal + hover, Cards = explicit).
- **Degraded stays glanceable:** when a quota source drops to the stale OAuth fallback or data ages out, the cell shows a visible amber dot + "fallback · Nm old" — **never** hover-only (degraded-state requirement).
- **Subscription leverage** renders in its own dashed box **outside** the grid ("Claude usage at API list prices · not billed") → the honesty model is **structural**, not badge-dependent.
- **Header:** `balanze` + subtitle (`measured usage` / `by provider`) + freshness (`⟳ 14s`) + density toggle + an (inert in v0.3.0) `⚙` gear.

Provenance badge vocabulary (Cards + tooltips): `REAL $` (solid `--real`), `QUOTA %` (outline), `EST` (dashed), `N/A` (dashed faint).

## 3. Architecture — reuse the actor spine

No new domain logic in `src-tauri`; it stays glue (boundary #8). The host fills in the existing `TauriSink` skeleton and boots the same composition `balanze-cli --watch` already runs.

```
JSONL / statusline / openai / codex / oauth
        │  (watcher tasks — boundary #4)
        ▼
   StateMsg::Update ──► StateCoordinator (owns Snapshot; boundary #7)
        ▲                   │  on every merge
        │ StateMsg::Query   ├──► TauriSink::on_snapshot ──► app.emit("usage_updated", &Snapshot)
   get_snapshot             │                          └──► tray repaint (ColorBucket, deduped §3.1)
   refresh_now ─► Refresh   └──► TauriSink::on_degraded ─► app.emit("degraded_state", DegradedPayload)
                                                          └──► tray warning overlay
```

**Boot sequence (in `src-tauri` `setup()`):**

1. Build the `StateCoordinator` with `TauriSink::new(app_handle)` as its `Sink`.
2. `app.manage(coordinator_handle)` so commands can reach it via `tauri::State`.
3. `Watcher::spawn(app_handle/coordinator_handle, &settings)` → retain the labeled `JoinHandle`s; supervise under `tokio::select!` (a panic / `Ok(Err(_))` is fatal; `Ok(Ok(()))` is graceful per boundary #4).
4. Tray + single-instance as today (don't refactor the `tray_by_id("main")` handler — see AGENTS.md troubleshooting).

**`TauriSink` (boundary #7 — the only tray caller):** fill the two TODO bodies in `src-tauri/src/tauri_sink.rs`:
- `on_snapshot(&Snapshot)` → `app.emit("usage_updated", snapshot)`; compute `(ColorBucket, title_text)`; if changed vs `last_painted`, `tray.set_icon(...)` (+ `set_title` on macOS) and update `last_painted`; else no-op.
- `on_degraded(Source, &str)` → `app.emit("degraded_state", DegradedPayload { source, error })`; flip the tray to the warning variant regardless of bucket.

## 4. IPC contract for v0.3.0

Frontend ↔ backend, Tauri commands/events only (boundary #9). Commands return `Result<T, String>`.

| Kind | Name | Shape | Notes |
|---|---|---|---|
| Command | `get_snapshot` | → **raw `Snapshot`** | First paint. Serde of `state_coordinator::Snapshot`, **not** the CLI `json_output` DTO. |
| Command | `refresh_now` | → `()` | Triggers an immediate refresh (mechanism — see §10 open items). |
| Event | `usage_updated` | **raw `Snapshot`** | Emitted by `TauriSink::on_snapshot` after every merge. |
| Event | `degraded_state` | `DegradedPayload { source: String, error: String }` | Per-source staleness/failure. |

**Decision (confirmed): emit the raw `Snapshot`.** The frontend mirrors the serde shape in TS and **owns popover presentation** (worst-window selection, pace-verdict text, provenance labels, µ-USD→display). No new shared DTO crate for MVP. Accepted cost: the small source→confidence mapping then exists in both `crates/balanze_cli/src/json_output.rs` and the frontend — low divergence risk (the mapping is small and stable); extract to a shared presentation crate later only if it grows. `Snapshot` and all nested types already derive `Serialize` — no backend serialization work.

## 5. Data-model → UI mapping

The contract the UI is built against. (`Snapshot` fields per `crates/state_coordinator/src/snapshot.rs` and the provider crates.)

| UI element | Source on `Snapshot` |
|---|---|
| Header freshness `⟳ 14s` | `fetched_at` (relative to now) |
| **Anthropic quota** headline % + tone | **5-hour** window util — preferred from `claude_statusline.payload.rate_limits.five_hour.used_percent` (live), fallback `claude_oauth.cadences[key="five_hour"].utilization_percent`. tone via thresholds (see §6). |
| …bar fill / tick | `pace[key="five_hour"]`: `used_fraction` (fill), `elapsed_fraction` (tick) |
| …reset clock | `…five_hour.resets_at` (relative) |
| …secondary `7-day 48%` | the 7-day window util (statusline/oauth) |
| …provenance tooltip | source = `statusline (live)` if statusline present, else `oauth (fallback)`; degraded if `claude_statusline_error`/`claude_oauth_error` or stale |
| **OpenAI quota** % + tone | `codex_quota.primary.used_percent` (+ `secondary`) |
| …bar tick | **frontend-computed** `elapsed_fraction` from `codex_quota.primary.{resets_at, window_duration_minutes}` and now (codex has no `pace` entry) |
| …reset / secondary | `codex_quota.primary.resets_at` / plan label |
| **Anthropic billed** | if `claude_oauth.extra_usage.is_enabled` → `$used/$limit overage` (`REAL`, `extra_usage_billed`); else **`unavailable`** hatched (`N/A`, "no per-user API spend") |
| **OpenAI billed** | `openai.total_usd` → `$X.XX` (`REAL`, `openai_admin_costs`); `openai_error` → "✗ fetch failed" |
| **Leverage box** | `anthropic_api_cost.total_micro_usd` when `total_event_count > 0` (`EST`, "list prices · not billed"); surfaces `anthropic_api_cost_error` / `claude_jsonl_error` here (the measured-only cells can't) |
| **Burn number** | `claude_jsonl.window.recent_burn_tokens_per_min` → "burn ~3.2k/min", or "—" when `None` (sparkline glyph deferred to v0.3.3) |
| **Cards** per-window bars | each `claude_oauth.cadences[]` (Anthropic) / `codex_quota.primary` (OpenAI) → one `UsageBar`; elapsed tick joined from `pace[key]` by key (Anthropic) / computed (codex) |

**Headline vs tray-color policy:** Grid Anthropic headline = the **5-hour** window (most actionable). The **tray `ColorBucket` = worst (max util) across all windows** of all connected providers ("worst window drives color", per wireframe + §3.1). These differ intentionally.

**Pace-per-cadence caveat:** Cards needs an `elapsed` tick for every cadence (incl. per-model 7-day). Confirm `window::pace` emits a `WindowPace` per cadence key; if per-model variants are absent, those Cards bars render used-only (no tick) as an accepted fallback. (Resolve in the plan.)

## 6. Tray gauge

- **`ColorBucket`** (`Green`/`Yellow`/`Orange`/`Red`) from the worst-window utilization; thresholds defined in `tauri_sink`/`tray_icon` (proposed: <50 Green, <75 Yellow, <90 Orange, ≥90 Red — tune during impl). A **stale/degraded** variant overrides the bucket so the user always sees "something's off".
- **Repaint dedup** by `(ColorBucket, title_text)` — only call `set_icon`/`set_title` when the tuple changes (§3.1). The 30s ticker sends `StateMsg::Refresh`; it never touches tray state directly.
- **Icon strategy (confirmed): render the ring gauge to RGBA at runtime** (`tauri::image::Image` from bytes) rather than a pre-rendered asset set — avoids asset sprawl and supports a continuous fill. Raster approach (hand-rolled vs a small pure-Rust raster crate like `tiny-skia`) is an open sub-decision (§10); a new dep needs the §8 gate.
- **Cross-platform:** macOS `NSStatusItem` can carry a text title (`set_title`); the **Windows tray is icon + tooltip only** (no inline text). For two providers, the gauge is a single composite icon (worst-window color); detail lives in the popover/tooltip.

## 7. Popover window mechanics

Replaces the scaffold's plain `show_main_window`:

- Window config (`tauri.conf.json`): `decorations: false`, `skipTaskbar: true`, `alwaysOnTop: true`, `resizable: false`, hidden on launch, transparent as needed for the rounded card + caret.
- Tray **left-click** → position the window near the tray/cursor, `show()` + `set_focus()`. **Hide on blur** (`WindowEvent::Focused(false)` → `hide()`), so it behaves like a menubar popover.
- The popover **hugs its content** (capped max-height + internal scroll) — a one-provider popover is visibly shorter than a two-provider one.
- Per-OS positioning differs (macOS: under the menu-bar item; Windows: above the system tray / at cursor) — handled with a small platform branch (§10).

## 8. Adaptive states (0 / 1 / 2 providers)

- **Two providers:** the full Grid/Cards as designed.
- **Single provider** (only one connected): Grid drops the empty column (`1fr` instead of `1fr 1fr`); popover is shorter; footer shows a passive "+ connect another provider" hint.
- **Unconnected provider:** column shows a passive "not connected — configure via CLI" note (the in-app "Connect →" / dismiss flow is v0.3.1, needs settings + keychain).
- **Tray** collapses to a single gauge when one provider is connected.

---

## 9. Error / degraded handling

- Each source's `_error: Option<String>` slot + the `degraded_state` event drive a **per-cell stale marker** (amber dot + "fallback · Nm old") — last-known data is shown, never blanked (boundary: IPC errors surface as `degraded_state`; §3.2).
- `claude_oauth_error` with statusline present → quota stays live (statusline is primary); only when both fail is the quota cell marked stale.
- Cost-synthesis / JSONL errors surface on the **leverage** line (the measured-only matrix cells can't show them).
- Tray flips to the warning variant on any degraded source regardless of bucket.

## 10. Frontend structure (Svelte 5)

```
src/
  routes/+page.svelte                  mounts <Popover/>
  lib/
    ipc.ts                             invoke(get_snapshot|refresh_now) + listen(usage_updated|degraded_state)
    types/snapshot.ts                  TS mirror of state_coordinator::Snapshot (raw shape)
    stores/usage.svelte.ts             runes store: snapshot, degraded map, loading; wires IPC
    presentation/
      worstWindow.ts                   pick headline + tray-color window (pure)
      pace.ts                          paceVerdict(used,elapsed) → {ratio,text,tone} (pure)
      provenance.ts                    cell → {badge, source, confidence, tooltip} (pure)
      format.ts                        microUsd→display, relative-time, burn (pure)
    theme.css                          light/dark tokens via prefers-color-scheme
    components/
      Popover.svelte  Header.svelte  DensityToggle.svelte
      GridView.svelte  CardsView.svelte
      QuotaCell.svelte  BilledCell.svelte  UsageBar.svelte
      ProvenanceTooltip.svelte  LeverageBox.svelte  BurnIndicator.svelte
src-tauri/src/
  lib.rs            boot coordinator+watcher+sink; register commands; popover window + tray click/blur
  commands.rs       (new) get_snapshot, refresh_now
  tauri_sink.rs     fill on_snapshot/on_degraded
  tray_icon.rs      (new) ColorBucket + runtime gauge raster
```

## 12. Open sub-decisions for the plan

1. **Tray raster approach** — hand-rolled RGBA ring vs a pure-Rust raster crate (`tiny-skia`). New dep → §8 gate.
2. **`refresh_now` mechanism** — a refresh channel to the watcher tasks (immediate re-poll) vs coordinator-driven recompute. Define the minimal honest behavior.
3. **Per-OS popover positioning** — exact tray/cursor anchoring on macOS vs Windows.
4. **`window::pace` per-cadence coverage** — does it emit a `WindowPace` per per-model cadence? If not, Cards bars for those windows render used-only (accepted fallback).
5. **ColorBucket thresholds** — confirm the 50/75/90 cut points against real usage.

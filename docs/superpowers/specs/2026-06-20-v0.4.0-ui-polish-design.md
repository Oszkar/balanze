# v0.4.0 - UI polish (design)

Status: approved for planning (2026-06-20). Scope is the v0.4.0 sub-milestone of Phase 4 (Productization) in `docs/PRD.md`.

## Goal

Make the popover read as intentional and credible - "a screenshot stands on its own" (the Success-criteria bar) - without drifting from polish into redesign. The visual delta is deliberate refinement on the existing Inter-based look, not a new identity. This pass also feeds the v0.4.2 release screenshots.

Grounding reference: the lo-fi wireframes in the `claude.ai/design` project (`wf-popover-main`, `wf-states`, `wf-prims`, `wf-settings`). They fix layout, states, and the provenance treatment; they are font-agnostic and are reference-only, not a required design. The `_ds/norrix-*` token bundle in that project belongs to an unrelated product and is ignored.

## Decisions (locked)

- **Identity:** keep Inter. Add `tabular-nums` for numeric cells and a deliberate type scale. No new font dependency (a tray popover should not pay font-loading cost).
- **Token depth:** add spacing + type scales to `theme.css` and migrate components off magic px. A small token set, not a full design system (~14 small components; a component framework would be YAGNI).
- **States:** build all four - loading/cold-start, empty + Connect CTA, error/stale, dismiss-to-hide.
- **Layout:** adopt row-label gutter, content-hugging popover height, and a `SettingsView` polish. (Header refresh-age was considered and dropped.)
- **Mockup:** none. Plan then iterate in `bun run tauri dev` with before/after screenshots.

## Scope

### In

1. **Token layer (`src/lib/theme.css`).** Add a 4px spacing scale (`--sp-1 .. --sp-8`) and a type scale (`--text-2xs .. --text-xl`, spanning the ~9.5-22px values already in use). Add `font-variant-numeric: tabular-nums` to numeric cells (quota %, $, pace, burn). Migrate component inline px to the tokens. Keep the existing light/dark color tokens; fill obvious gaps only.
2. **Provenance badges (`Badge.svelte`).** Align to a 4-kind taxonomy so the matrix's honesty reads at a glance:
   - `REAL $` - solid `--real` background, white text (real billed money).
   - `QUOTA %` - outlined `--ink` (server-authoritative quota).
   - `EST` - dashed `--faint` (the leverage / estimate tier).
   - `N/A` - dashed faint (honestly unavailable).
3. **States.**
   - **Loading / cold-start** - replace the bare "connecting..." text with a proper skeleton/affordance for the first-fetch window (OAuth back-off cold start).
   - **Empty + Connect CTA** - for an expected-but-unconnected provider, a "Connect ->" card (with a "paste admin key" hint) that routes into Settings. Mirrors `wf-states` `PopEmpty`.
   - **Error / stale** - one consistent failed-fetch / stale cell treatment, tied to the existing `degraded_state` banner rather than ad-hoc per-cell strings.
   - **Dismiss-to-hide** - an `x` on a provider column disables it via the existing `set_settings` IPC (`providers.{openai,codex}_enabled = false`); the popover drops to the single-provider view; "re-add in Settings" re-enables. No schema change.
4. **Layout.**
   - **Row-label gutter** - `GridView` grid goes `1fr 1fr` -> `46px 1fr 1fr` with a left Quota % / Billed $ label column, so the matrix is self-explanatory.
   - **Content-hugging height** - the popover sizes to content (a one-provider popover is visibly shorter than a two-provider one). See "Backend change" below.
   - **Settings polish** - regroup `SettingsView` into per-provider connector cards (status dot + inline masked key field + toggles), tokenized. Keep the existing IPC and behavior.
5. **Hygiene** - sweep the em-dash in `GridView` (`'—'` fallback) and any others per AGENTS 3.5; apply `tabular-nums`; normalize spacing to the new tokens.

### Out (deferred per phasing)

- Burn **sparkline** / trends - v0.6 dashboard, needs durable history.
- In-app **setup wizard** - a feature, not polish (the `wf-settings` `SettingsWizard` frame); revisit with the CLI/onboarding story.
- **Alerts** toggle - v0.5.
- **Header refresh-age** indicator - considered and dropped for v0.4.0.

## Backend change (the one host touch)

Content-hugging height is not pure CSS: the popover is a fixed-height Tauri window (`src-tauri/tauri.conf.json` `height: 560`), and `src-tauri/src/lib.rs` anchors it to the tray/dock using the window height (notch-/DPI-aware). Design:

- **Frontend** observes its content height (a `ResizeObserver` on the popover root) and calls a new IPC command `resize_popover(height: u32)`. The webview keeps **zero** window-mutation capability - it asks via this command, exactly like the existing `hide_window`.
- **Host** (`commands.rs` + `lib.rs`): `resize_popover` clamps the requested height to a sane min/max, sets the window inner size, then re-runs the existing anchor positioning so the popover stays pinned to the tray/dock line after the resize.
- **IPC contract:** `resize_popover` is a new command on the surface, so the IPC table in `docs/ARCHITECTURE.md` is updated in the same PR (AGENTS 2.1: IPC surface changes need a doc update).
- **Capability:** no new Tauri capability for the webview (the command is the boundary). If the host code needs a window API, that stays Rust-side.

Risk: multi-monitor / fractional-DPI / the notch-aware anchor math. Mitigation: clamp height, reuse the existing anchor function unchanged (only its inputs move), and a manual multi-monitor + DPI pass before merge.

## Non-goals / guardrails

- No redesign, no new font, no component framework, no new crate.
- No `UsageEvent` / `Snapshot` / `Settings` schema change. Dismiss reuses `providers.*_enabled`.
- Avoid drive-by refactors (AGENTS 5). Touch a component only to tokenize it, restyle it, or add a state it owns.

## Testing & validation

- **Presentation logic** stays unit-tested with vitest. Extend the existing `format` / `quota` / `pace` specs for any new pure helpers (e.g. refresh-age formatting if added, state-selection predicates). Pure helpers get a test before the impl change (AGENTS 6, for pure modules).
- **Visual** verification in `bun run tauri dev`: each state (loading, connect-CTA, error/stale, dismissed, single-provider, two-provider), light + dark, grid + cards. Capture before/after screenshots.
- **Content-hugging** gets a manual multi-monitor + DPI check (the one host-touching item).
- Gates: `bun run check`, and for the `src-tauri` touch the standard Rust matrix (`cargo build`/`clippy`/`nextest`/`fmt`) plus the `tauri dev` smoke (tray appears, click opens, Quit exits).

## Open implementation details (for the plan)

- Exact type-scale step values and which components map to which step.
- The precise state machine: when a provider column shows the Connect-CTA vs hides entirely vs shows data, across enabled/disabled x configured/unconfigured x error.
- Where the dismiss `x` lives in grid vs cards view, and the "re-add in Settings" affordance copy.
- Content-hugging clamp bounds (min/max height) and debounce of the `ResizeObserver` -> `resize_popover` calls.

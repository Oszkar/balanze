Balanze / cross-surface reference

One color language.
Four surfaces, one reading.

Balanze shows the same usage on four surfaces - the tray icon, the desktop popover, the CLI, and the Claude Code statusline. This page is the reference for what they share: one green / yellow / orange / red scale at 50 / 75 / 90 on every gauge, one glyph per provider on the statusline, and a tray whose number can never contradict its color.

The scale

Every utilization gauge uses green / yellow / orange / red at 50 / 75 / 90. One window::Severity classifier owns the cutoffs; the tray, CLI, and statusline import it, and the Svelte popover mirrors it.

The tray

Ring, title, and tooltip derive from one view, so a red icon can't sit beside a low number. A header names the worst window and explains cold-start / degraded states.

The surfaces

All four share the scale and the vocabulary. They differ only in depth: a glance surface (tray) shows less than a detail surface (popover, CLI).

01The severity scale

Read any vertical slice: all four surfaces land in the same band. The cutoffs are inclusive lower bounds - exactly 50 is yellow, 75 is orange, 90 is red - and every surface classifies the rounded displayed value, so 89.6 shows "90%" and reads red.

02The tray at a glance

The ring colors on the worst window across providers. The title and tooltip keep a stable layout - each available provider in its own fixed slot, an unconfigured one omitted - and a header names what is worst, so the color is always explained by a number you can see.

Title (macOS menu bar) + icon ring

Each available provider sits in its own fixed slot, showing its worst window (an unconfigured one is omitted). The ring matches the larger of the two shown numbers - here the red comes from Claude's 94%.

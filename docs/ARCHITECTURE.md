# Architecture

Balanze is an actor-model Tauri app. One thread owns canonical state; pollers feed it; the frontend reads it. Currency is `i64` micro-USD; HTTP is concentrated in two clients; secrets route through one keychain wrapper; the JSONL wire format is owned by one crate. The frontend ‚Üî backend surface is a fixed set of commands and events.

This doc is the architecture reference. Product scope lives in [`PRD.md`](PRD.md); operational discipline (validation gates, change control, test rules) lives in [`../AGENTS.md`](../AGENTS.md).

## Boundaries

Strict layering - agents must respect these.

1. **`claude_parser` owns the JSONL wire format.** Other crates consume `UsageEvent` only; field names and line-format quirks do not leak. The incremental/tailing reader (`IncrementalParser`) skips an unparseable *complete* line - a WARN with the skipped count - and advances its byte cursor past the batch, so one malformed or schema-drifted line cannot stall future appends for that file; the strict batch parser (`parse_str`) still fails fast for callers that want that.
2. **`window`, `claude_cost` are pure functions.** No I/O, no `tokio::spawn`, no logging above `debug`. `claude_cost` is infallible - unknown models route to `skipped_models`.
3. **`anthropic_oauth` + `openai_client` are the only HTTP clients.** Other crates do not issue HTTP requests; the glue crates (`balanze_cli`, `watcher`) may construct a `reqwest::Client` to inject into these two, but all request/retry/redaction logic lives here. `anthropic_oauth` is additionally the only reader of Claude Code's OAuth credential. Both `~/.claude/.credentials.json` / `~/.config/claude/.credentials.json` and the recent macOS login Keychain representation belong to Claude Code and are strictly read-only to Balanze. File credentials are re-read each poll to observe Claude Code's atomic replacements; the Keychain credential is read via `/usr/bin/security` (not `keyring`, so boundary #5 holds) and cached while valid to avoid repeated prompts. Balanze never calls the token-refresh endpoint and never writes, mirrors, persists, or backs up credential contents. An expired or rejected credential surfaces `CredentialExpiredReadOnly` (re-run `claude login`). A future explicit file-refresh opt-in is tracked in [#186](https://github.com/Oszkar/balanze/issues/186) and requires a separate settings-schema and secret-boundary change.
4. **`watcher` owns `notify` + the debounce + the 60 s safety poll.** Other crates do not import `notify`. `Watcher::spawn(handle, &Settings) -> Vec<(&'static str, JoinHandle<...>)>` returns one labeled handle per task; default-spawned (3): `jsonl` (notify + 300 ms debounce + a 60 s fallback; reads new bytes only via a per-file byte cursor, `claude_parser::IncrementalParser`, so neither path full-reparses after launch per ¬ß3.1), `statusline`, `safety` (a 60 s re-read of statusline + Codex - it no longer scans JSONL, the `jsonl` task's 60 s fallback covers that; its Codex scan gated on `providers.codex_enabled`); conditionally (2): `openai_poll` when `providers.openai_enabled` or a `BALANZE_OPENAI_KEY` env override is set, and `oauth_poll` when `providers.anthropic_enabled`. The caller (`balanze-cli watch`, or the Tauri host) supervises under `tokio::select!`. `Ok(Ok(()))` is graceful (e.g. no OpenAI key) and must not trigger teardown. The CLI treats an `Ok(Err(_))` / panic from any task as fatal and exits with a restart hint. The Tauri host self-heals instead: it re-spawns the whole set on a settings change (provider toggles apply live) AND on an unexpected task death, where it also emits a `degraded_state` for the affected source (mapped by `watcher::source_for_label`) and backs off (exponential, 1 s to a 60 s cap) so a persistent failure stays visibly degraded rather than silently frozen. Tasks feed the coordinator directly via `StateCoordinatorHandle::send(StateMsg::Update(...))`.
5. **`keychain` is the only caller of `keyring-core` (plus the OS-native store crates).** All secret reads/writes route through this crate.
6. **`settings` owns `settings.json`.** Atomic writes (tmp + rename). No other crate reads or writes this file.
7. **`state_coordinator` is the only writer of the in-memory `Snapshot` and the only driver of tray repaints (via the `Sink` trait).** Pollers send `StateMsg::Update(SourcePartial)`; the coordinator applies the partial and notifies the `Sink`. The concrete `TauriSink` (in `src-tauri`) is the only caller of OS tray APIs: it emits `usage_updated` and repaints, deduping by `(ColorBucket, title, tooltip)` (its own `last_painted`) - the tooltip is part of the key because it names the worst window (and now carries both Codex windows), so a same-color/same-title repaint that only changed the tooltip must still paint. A `StateMsg::Refresh` (popover open / `refresh_now`) re-notifies the `Sink` with current state so it can repaint - it never re-fetches, and nothing outside the `Sink` touches OS tray state. The `ClaudeJsonl` partial carries **raw deduped events**, not a finished summary: the coordinator derives the window + API-rate cost via `state_coordinator::summarize_jsonl`, anchoring the rolling window to the latest OAuth `resets_at`. It caches the events, so a subsequent OAuth update (a new reset) re-anchors them - this is what keeps the live `watch` window identical to the one-shot CLI. It also handles `StateMsg::SettingsChanged`: on a provider toggle it resets that provider's cell to "not configured" (so disabled providers don't linger with stale data) and repaints; the watcher re-spawn (boundary #4) stops/starts the actual polling. A `StateMsg::SourceUnavailable { source, reason }` marks a source intentionally not-configured (today only `ClaudeOAuth`, set by `oauth_poll` when Claude Code isn't installed): it sets a neutral `claude_oauth_unavailable` marker, drops any stale data/error + pace, and repaints via the `Sink` - distinct from an `Update(Err)`, so the tray stays on the neutral (not warning) bucket and no `degraded_state` is emitted; a later successful `Update` clears it. The host wraps its `Sink` in `state_coordinator::SnapshotFileSink`, which atomically writes the full `Snapshot` to `<ProjectDirs.data>/snapshot.json` on each update (best-effort; a write failure logs and is swallowed, never breaking the inner sink) for `balanze-cli statusline` to read - the reverse direction of the statusline.snapshot.json bridge (boundary #12), with zero statusline-initiated network on the read path.
8. **`src-tauri` and `balanze_cli` are glue, not logic.** Both turn backend crates into a `Snapshot`, by two routes: the **one-shot** path (`balanze-cli status`) runs `snapshot_composer::compose`; the **live** path (`balanze-cli watch`, or the Tauri host) feeds incremental `StateMsg::Update` partials into the coordinator, which assembles the `Snapshot`. The JSONL ‚Üí window+cost math is shared across both routes via `state_coordinator::summarize_jsonl` (called by `compose` and by the coordinator merge), so the live path cannot diverge from the one-shot path on the window anchor. Identical inputs ‚áí identical `Snapshot` cells (parity-tested in `integration_4quadrant::compose_parity_against_fixtures`).
9. **Frontend ‚Üî backend goes through the fixed IPC contract.** Commands: `get_snapshot`, `get_history`, `refresh_now`, `hide_window`, `resize_popover`, `set_api_key`, `validate_api_key`, `has_api_key`, `clear_api_key`, `get_settings`, `set_settings`, `get_statusline_status`, `set_statusline_wired`, `replace_statusline`, `restore_statusline`. Events: `usage_updated`, `degraded_state`. Adding to this surface needs a design-doc update first.
10. **Currency math uses `i64` micro-USD.** Convert to `f64` only at the display boundary. Every provider's money is converted to i64 micro-USD at its own ingest/parse boundary (Anthropic `extra_usage` cents, OpenAI Admin Costs dollars), so the `Snapshot` holds no `f64` money. `claude_cost` keeps per-token prices in i64 nano-USD with i128 intermediates and saturates at `i64::MAX`.
11. **`codex_local` knows the Codex rollout-JSONL format and is the only reader of `~/.codex/`.** Honors `CODEX_CONFIG_DIR`. Other crates consume `CodexQuotaSnapshot`, which carries both rolling rate-limit windows (5h + weekly, classified by duration, never by which JSON slot they land in); a consumer that needs a single "Codex %" figure reduces the two to the worst via `CodexQuotaSnapshot::worst_window`.
12. **`claude_statusline` owns the Claude Code statusLine wire format and the statusLine stanza in Claude's `settings.json`** (read + atomic write, idempotent, no-clobber - mirrors boundary #3). Also owns the `<ProjectDirs.data>/statusline.snapshot.json` inter-process bridge: `balanze-cli statusline` writes atomically; `watcher::tasks::statusline` reads on notify. (The reverse-direction `snapshot.json` bridge - host writes the full cross-provider `Snapshot`, `balanze-cli statusline` reads it for the Codex/OpenAI segments - is owned by `state_coordinator`, boundary #7.)

## IPC contract

Frontend ‚Üî backend, via Tauri commands and events only. Commands return `Result<T, String>` (`anyhow::Error::to_string()`).

| Direction | Name | Purpose |
|---|---|---|
| Command | `get_snapshot` | Current `Snapshot`. |
| Command | `get_history` | Recent rolling-window-sized history. |
| Command | `refresh_now` | Trigger an immediate poll/refresh. |
| Command | `hide_window` | Hide the popover (ESC-to-dismiss); window manipulation stays in Rust. |
| Command | `resize_popover` | Resize the popover window to a content-hugging height (clamped) and re-anchor to the tray/dock. Frontend-driven via `ResizeObserver`; webview holds no window capability. |
| Command | `set_api_key` / `has_api_key` / `clear_api_key` | Store / probe-presence / remove the user-supplied OpenAI key in the keychain (never returns the value). |
| Command | `validate_api_key` | Probe a user-supplied key against the provider WITHOUT storing it (async; one fail-fast month-to-date costs request, the same call the poller makes). Returns `{ ok, retryable, message }` so Settings gives immediate feedback: `ok` authenticated, `retryable` failed transiently (network / 429) so the UI may offer "save anyway", neither means the key is definitively wrong. Never logs or echoes the key. |
| Command | `get_settings` / `set_settings` | Non-secret config (settings.json shape). `set_settings` live-applies provider toggles. |
| Command | `get_statusline_status` / `set_statusline_wired` | Read (returns `{ status, command, replaced_command }`) / wire-unwire the Balanze `statusLine` in Claude Code's `settings.json` (no-clobber). |
| Command | `replace_statusline` | Replace a foreign `statusLine.command` with Balanze's, backing the displaced command up to `settings.statusline.replaced_command` before writing. Rolls back the backup if the wire step fails. Only Claude Code's `settings.json` (the wire) and Balanze's own `settings.json` (the backup) are written - the foreign tool's own config files are never read or modified. |
| Command | `restore_statusline` | Write `settings.statusline.replaced_command` back as `statusLine.command` (or unwire if the backup is `None`), then clear the backup. Provider-agnostic. |
| Event | `usage_updated` | New `Snapshot` available. |
| Event | `degraded_state` | A source is stale / errored; surface visually. |

**Status:** implemented: `get_snapshot`, `refresh_now`, `hide_window`, `resize_popover`, `set_api_key`, `validate_api_key`, `has_api_key`, `clear_api_key`, `get_settings`, `set_settings`, `get_statusline_status`, `set_statusline_wired`, `replace_statusline`, `restore_statusline`, `usage_updated`, `degraded_state`. Everything except `get_snapshot` / `refresh_now` / `validate_api_key` is a synchronous command (Tauri runs them off the async runtime); `validate_api_key` is async because it makes a network request (it delegates to `watcher::validate_openai_key`). The key commands touch only the keychain (the user-supplied OpenAI key; Anthropic uses Claude Code's OAuth) and never return the value; `set_statusline_wired` delegates to `claude_statusline` and is no-clobber. `replace_statusline` and `restore_statusline` are synchronous commands that back up / restore a displaced foreign `statusLine.command` in Balanze's own `settings.json` (`statusline.replaced_command`); the only files written are Claude Code's `settings.json` (the wire) and Balanze's own `settings.json` (the backup) - no foreign tool's own config file is ever read or modified. Restore refuses to overwrite a foreign command that Balanze did not displace (it keeps the backup instead). `set_settings` / `set_api_key` / `clear_api_key` live-apply: they send `StateMsg::SettingsChanged` (coordinator clears disabled cells) and signal the host to re-spawn the watcher, so provider toggles take effect without a restart. Planned: `get_history` (the durable local history it reads is deferred - revisited post-1.0 with the dashboard; the v0.4.x CLI `export` re-derives the series from JSONL on each run and persists nothing, so `get_history` stays planned, tied to whenever storage lands). `refresh_now` today re-emits the current snapshot to the sink (repaint + catch-up); an on-demand provider re-poll is a later follow-up.

The CLI `--json` schema is the same `Snapshot` rendered through a presentation DTO (`crates/balanze_cli/src/json_output.rs`): the document carries a top-level `schema_version` (currently `1`) so a consumer can detect a breaking change, and every money cell is `{ value_micro_usd: i64, source, confidence, details }` - OpenAI Admin Costs included, converted to i64 micro-USD at the `openai_client` parse boundary so no money cell is `f64` - so consumers tell `jsonl_list_price` / `estimate` apart from `openai_admin_costs` / `real` and `extra_usage_billed` / `real` from the wire shape. Two extra cells (v0.2): `claude_statusline` carries the live `StatuslineFilePayload` envelope (Claude Code's session estimate - a distinct cost tier, no money normalization); its `rate_limits` block carries `five_hour`/`seven_day` (unchanged position) plus a `windows` array with every rate-limit window the statusLine payload reported, mirroring `claude_oauth.cadences` - a statusLine-sourced snapshot is never capped at the two named windows. `.pace` carries a per-window array (`key`, `used_fraction`, `elapsed_fraction`, `ratio`) derived from the OAuth cadence bars - used % vs elapsed % of each quota window (5h, 7d) plus their ratio, computed by `window::pace`; `ratio` is null right after a window reset. `watch --json` reuses this DTO, one JSON document per line. Identifiers (`org_uuid`, Codex `session_id`) are redacted unless `-v`.

**On-disk IPC files.** Two atomic JSON files bridge processes (distinct from the Tauri command/event surface above), flowing in opposite directions:

| File (`<ProjectDirs.data>/`) | Writer | Reader | Payload |
|---|---|---|---|
| `statusline.snapshot.json` | `balanze-cli statusline` (every turn) | `watcher::tasks::statusline` (on notify) | `StatuslineFilePayload` (`schema_version` 2) - Claude session estimate + a generic rate-limit window list (`RateLimits.windows`, not just five_hour/seven_day; boundary #12) |
| `snapshot.json` | `state_coordinator::SnapshotFileSink`, from whichever host owns the coordinator (the Tauri app or `balanze-cli watch`) | `balanze-cli statusline`, when fresh (within 120 s of `captured_at`) | `SnapshotFilePayload { schema_version, captured_at, snapshot }` - the full cross-provider `Snapshot`; fills the Codex quota + OpenAI billed-$ statusline segments (boundary #7) |

The statusline does no network on the `snapshot.json` read path; Claude segments always come from stdin (the no-arg `balanze-cli statusline` contract is frozen - Claude Code invokes it once per turn, piping the session payload). An additive `statusline restore` subcommand exists (restores the backed-up foreign statusLine or unwires; it does not read stdin; the no-arg + stdin contract is unchanged). An absent or unreadable file degrades to a Claude-only line. Both files use the same atomic tmp+fsync+rename discipline and a `schema_version` probe; neither holds secrets.

**Self-compose fallback cache.** When no fresh `snapshot.json` exists (no desktop app or `balanze-cli watch` running), `balanze-cli statusline` self-composes the cross-provider segments: Codex from local files and OpenAI from the Admin Costs API via a short-timeout (3s) client. The OpenAI result is persisted to `<ProjectDirs.cache>/statusline/openai-cost.json` (or `<BALANZE_CACHE_DIR_OVERRIDE>/statusline/openai-cost.json` when that env var is set) so the upstream API is called at most once per 300 seconds - this is the AGENTS.md ¬ß3.1 OpenAI politeness gate. The cache entry is keyed by an FNV-1a fingerprint of the resolved OpenAI key (never the key itself - AGENTS.md ¬ß3.4); a 300s negative-cooldown (matching the TTL) prevents retry-hammering a broken endpoint, so a failing endpoint is also polled at most once per 5 minutes; stale-while-updating keeps the segment populated rather than blank. The self-compose path calls only `codex_local` and `openai_client`; it never reaches `anthropic_oauth` (the AGENTS.md ¬ß3.1 politeness invariant). Source precedence: a fresh `snapshot.json` (within 120s) wins whole; otherwise the self-composed cross-provider cells are merged over the stale snapshot per cell (each cell comes from self-compose when present, else the stale snapshot cell with its staleness marker), so a last-known value stays visible when only one source has it; else Claude-only.

## Errors and degraded state

- App-level results use `anyhow::Result<T>`. No `.unwrap()` outside tests.
- `thiserror` enums live only in `claude_parser` - variants include `FileMissing`, `IoError`, `SchemaDrift { line, message }` so the StateCoordinator can pattern-match and set the right `DegradedState`.
- Long-running tasks (file watcher, polling, the coordinator itself) are supervised with a retained `JoinHandle` + `tokio::select!`. A coordinator exit/panic is fatal on both hosts (the CLI returns; the Tauri host calls `AppHandle::exit` on a genuine panic, but not on a shutdown abort). A watcher-task `Ok(Err)` / panic is fatal in the CLI (`watch` exits with a restart hint); the Tauri host surfaces a `degraded_state` for the affected source and re-spawns the set with bounded backoff (self-heal, not silent freeze).
- External I/O retries via `backoff` (`standard` = 30 s √ó 2‚Åø cap 10 min; `fail_fast` = 0). Idempotent GETs retry on 429 + 5xx + transport; the token-rotating refresh POST retries 429-only (a replayed consumed refresh token strands the user).
- IPC errors surface to the UI as a `degraded_state` event; the tray icon shows a warning dot.
- "Unavailable / not configured" is a **neutral** state, distinct from degraded: a source whose prerequisite is absent (Claude Code not installed, so no OAuth credential) reports `StateMsg::SourceUnavailable`, which sets `Snapshot::claude_oauth_unavailable` and repaints without an error - the tray shows the neutral bucket (not the warning dot) and the cell reads "not configured" rather than spinning on a loading state. Only `ClaudeOAuth` carries this marker today.
